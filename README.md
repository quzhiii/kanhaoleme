# 看好了么 / Kanhaoleme

看好了么是一个 AI 驱动的真实就医与院外健康体验社区，用“小看”AI 向导帮助用户查找、整理和分享真实经历，让看病前的信息更透明、更轻松、更可信。

> Demo only. Not medical advice.

## 项目解决的问题

- 看病前信息不透明，用户很难提前了解真实流程、等待、费用线索和沟通体验。
- 真实经验分散在小红书、群聊、论坛和问答平台，检索成本高。
- 医疗 AI 擅长知识解释，但缺少真实用户连续体验和院外过程信息。
- 泛内容平台广告混杂、来源难辨、费用状态不清，用户很难判断内容可信度。

## 核心功能

- 小看 AI 社区向导：帮助用户从自然语言问题进入相关经历。
- 真实经历卡片：把流程、费用线索、阶段、城市和可确认问题结构化呈现。
- 来源可信标识：区分用户自述、AI整理、费用自述、费用已核验、官方资料、专业审核中、商业内容和高风险待审。
- 费用线索：只展示用户自述或有限核验的历史线索，不做报价或报销承诺。
- 分享经历预览：用户可以先填写经历并查看预览，当前 Demo 不提交、不保存、不审核。
- 城市圈子占位：为未来接入家庭医生、社康和医联体公开服务信息预留空间。
- 合规边界说明：所有页面强调平台不提供诊断、治疗方案、用药建议或机构推荐。

## AI Agent 设计

当前 Demo 展示的是 Agent 产品设计和前端交互原型，不包含真实后台 Agent 调用。

- 前台小看向导：查找相似经历、解释来源状态、整理可向正规机构确认的问题。
- 脱敏 Agent：未来用于识别手机号、住院号、完整病历、票据、详细地址等敏感信息。
- 结构化 Agent：未来把自由叙事整理成体验卡字段。
- 来源识别 Agent：未来区分用户自述、官方资料、商业内容和费用状态。
- 风险审核 Agent：未来识别医疗建议、疗效承诺、广告和高风险健康表达。
- 本地服务 Agent：未来整理城市公开服务信息，不做机构推荐或付费排序。

## 当前 Demo 状态

- 当前是静态前端 Demo。
- 当前使用 18 条 mock 数据，不代表真实用户。
- 当前不处理真实用户数据。
- 当前不接入真实医疗 AI。
- 当前没有后端、登录、数据库、真实投稿、审核后台、真实检索、媒体上传、评论互动或持久化数据。
- 当前不提供医疗建议，不代表正式上线产品。

## 安全与边界

看好了么展示的是用户真实体验、AI 整理信息和公开资料，不提供诊断、治疗方案、用药建议或费用承诺。具体健康问题请咨询正规医疗机构或专业人员。

- 不诊断。
- 不治疗。
- 不荐药。
- 不排名医生。
- 不推荐机构。
- 不做费用承诺。
- 不让小看冒充医生、病友或真实患者。
- 费用只是用户自述或有限核验线索。

## Roadmap

- v0.2 Demo hardening：补齐筛选、可编辑向导、投稿安全检查和公开文档。
- 20-30 人内测：验证用户是否理解可信标识、是否愿意投稿、是否误认为问诊。
- 受控社区 MVP：引入登录、草稿、真实投稿、审核状态和删除撤回机制。
- 审核后台：支持人工审核、风险原因码、举报和审计日志。
- API 和数据库：在授权、脱敏、删除撤回和合规机制准备好后接入。
- 城市服务信息：整理家庭医生、社康和医联体公开服务信息。
- B2BC 合作验证：验证非广告型机构公开回应和匿名聚合体验报告。

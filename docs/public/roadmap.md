# Kanhaoleme Public Roadmap

Date: 2026-07-02

## Current Stage

Kanhaoleme is currently a v0.2 static frontend demo for 0-1 validation. It uses mock data and does not process real health data.

## Phase 1: Demo Hardening

- Fix build and lint gates.
- Keep all mock content clearly labeled as demo content.
- Add topic, city, and stage filters to the experience list.
- Make the XiaoKan guide input editable while keeping it local and non-diagnostic.
- Improve the share preview page with sensitive-information checks and disabled submit explanation.
- Add public architecture, roadmap, safety, and demo-script docs.

## Phase 2: Internal Testing

- Run 20-30 user interviews or guided tests.
- Validate whether users understand trust badges.
- Validate whether users see XiaoKan as a community guide rather than a doctor.
- Validate whether users are willing to share safe, de-identified experiences.
- Document misunderstanding points and revise copy.

## Phase 3: Invitation-Only MVP

- Add login or invitation-based access.
- Add draft saving and real submission flow.
- Add deletion and withdrawal paths.
- Add basic API and database contracts.
- Keep all health-related content review-first, not publish-first.

## Phase 4: Moderation Dashboard

- Add review queue.
- Add approve, reject, hide, and needs-edit actions.
- Add risk reason codes.
- Add reporting and audit logs.
- Ensure high-risk health content always requires human review.

## Phase 5: Real Submission And Community

- Add approved community feed.
- Add post detail pages.
- Add first-level comments after moderation.
- Add low-risk reactions such as “I had a similar experience.”
- Add topic groups created by the platform, not by users.

## Phase 6: City Circles

- Add public family doctor, community health center, and medical alliance information where available.
- Clearly separate official public information from user stories.
- Avoid paid ranking, provider recommendation, or diagnosis-like guidance.

## Phase 7: B2BC Validation

- Explore non-advertising provider response tools.
- Explore anonymized aggregate experience reports.
- Validate whether institutions value transparent, de-identified patient experience feedback.

## Deferred

- Real medical AI API.
- Report interpretation.
- Drug or treatment recommendations.
- Doctor or institution ranking.
- Paid placement.
- Cost prediction.
- Private messaging or group chats.
- Long-term companion NPC behavior.
- User-created groups.
- Video/audio upload.
